# Changelog

KIFF is built incrementally. Each "brick" is a small, focused addition to the framework that ships with tests and a working demo path. These notes explain what each brick added and why, in the order they landed.

If you are adopting KIFF for the first time, you do not need to read these. Start with the top-level [README](../../README.md), then [docs/build-a-domain.md](../build-a-domain.md). Come back here when you want to understand how a specific capability evolved or why a design choice was made.
